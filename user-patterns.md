# 사용자 작업 패턴

## 커뮤니케이션 스타일
- **응답이 극도로 간결함**: "응 진행해줘", "좋아", "o.k", "b" 같은 한두 글자 승인
- **질문 방식**: 옵션 제시 → 알파벳 선택(a/b/c), 또는 "어떤 걸 더 추천해?" 후 추천 수용
- **설명 불필요**: 왜 했는지 설명 말고 결과만 보여줄 것. 긴 요약 금지
- **한국어 우선**: 모든 대화·UI 텍스트는 한국어

## 디자인 심미관
- **레퍼런스 기반**: 항상 스크린샷·이미지 첨부 후 "이렇게 해줘"로 방향 제시
- **럭셔리 브랜드 톤**: 루이비통·고급 레스토랑 수준의 미니멀·프리미엄
- **컬러 원칙**: 에스프레소(`--color-espresso`) 주 포인트, 과한 색상 금지
- **아이콘**: 이모지 사용 금지 → 한자(材熟竹饌火) 또는 단순 SVG
- **타이포**: 트래킹·폰트 모노 디테일 중시, 유치한 요소 즉시 제거

## 수치 조정 패턴
- 결과물을 보고 나서 **비율로 조정 지시**: "1.7배 키워줘", "30% 줄여줘", "15% 확대"
- 한 번에 완성하려 하지 않음 — 보고·조정·보고 반복 루프

## 에셋(사진) 워크플로
- 사진은 **직접 프로젝트 폴더에 넣고** 위치를 알려줌 ("사진/육즙관리소 15에 넣어두었어")
- 에이전트는 해당 경로 확인 후 `public/` 하위로 복사
- KakaoTalk 공유 이미지(원본 파일명 유지)는 품질 확인 후 web-safe 파일명으로 변경

## 아키텍처 선호
- **DB 주도 콘텐츠**: 이미지 메뉴판보다 Supabase DB 기반 디지털 메뉴 선호
- **정적보다 동적**: 하드코딩 최소화, 데이터는 DB에서
- **master = 배포 기준**: 기능 완성 후 worktree → master 머지, 중간 상태 머지 금지

## 섹션 구성 원칙 (LP 기준)
```
Hero → 신뢰지표 → Why → 지점 → 인테리어배너 → 단체CTA → 브랜드스토리 → 리뷰 → SNS → 예약CTA
```
- 소셜 프루프(리뷰)는 항상 CTA 직전에 배치
